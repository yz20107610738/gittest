
1.git五分钟教程

/*  github官网 注释
    git init
    git add README.md
    git commit -m "first commit"
    git remote add origin https://github.com/yz20107610738/ss.git
    git push -u origin master
*/
// master是分支  origin是通道  一般来说通道只有一个，分支会有多个

初始化仓库。使用当前目录作为Git仓库。
git init

使用我们指定的目录作为git仓库
git init newrepo

- 添加新文件
git add filename
git add .   // 添加所有的文件

- 提交版本 
现在我们把它提交到仓库
git commit -m "Adding files"

如果您不使用-m，会出现编辑器来让你写自己的注释信息。
当我们修改了很多文件，而不想每一个都add，想commit自动来提交本地修改，我们可以使用-a标识。
git commit -a -m "Change some files"

- 发布版本
从服务器上克隆一个库
git clone ssh://~~

我们修改之后可以进行推送到服务器
git push ssh://~~

-取回更新
如果您已经按照上面的进行push, 下面命令表示，当前分支自动与唯一一个追踪分支进行合并
git pull 
从非默认位置更新到指定的url
git pull http://git.example.com/~~


-删除
如果你想从资源库中删除文件，我们使用rm
git rm file

要创建一个新的分支
git branch test

branch命令不会将我们带入分支，只是创建一个分支，所以我们使用checkout命令来更改分支
git checkout test
我们在分支上修改点东西

第一个分支，或主分支，被称为"master"。
git checkout master
git merge test

如果你想删除分支，我们使用 -d 标识
git branch -d -test
